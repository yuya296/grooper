# Requirement

- [ ] Purpose / value: 正規表現マッチ時にキャプチャ結果でグループ名を動的生成できるようにし、ドメイン/パス単位の自動分類を柔軟化する。
- [ ] Scope:
  - `matchMode=regex` のルールで `group` にキャプチャ参照を記述可能にする
  - サポート記法: `$1`, `$2`, ... と `$<name>`
  - `$$` は `$` エスケープとして扱う
  - 既存の `${var}`（vars展開）との共存ルールを定義して実装
  - Options UIの説明文とバリデーション更新（regex時のみキャプチャ展開有効）
- [ ] Out of scope:
  - globモードでのキャプチャ展開
  - `group` 以外のフィールドへのテンプレート展開
- [ ] Assumptions:
  - 展開順は `vars展開 -> regexマッチ -> capture展開` とする
  - 参照先が存在しないキャプチャは空文字に展開する
- [ ] Risks:
  - `${var}` と `$1` の混在で設定者が混乱する
  - 命名キャプチャのサポート差異による誤動作

# Acceptance Criteria

- [ ] AC1: `matchMode=regex` で `$1` / `$<name>` を使った `group` が期待通りに展開される。
- [ ] AC2: `matchMode=glob` でキャプチャ記法を使った場合、保存/検証時に明確なエラーを返す。
- [ ] AC3: 既存設定（キャプチャ未使用、`${var}` 使用含む）との後方互換が維持される。

# Notes

- 記法は「正規表現ライク」を優先し、`${...}` は vars専用として意味を分離する。
