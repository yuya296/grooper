# Requirement

- [ ] Purpose / value: GrooperをChrome Web Storeへ申請・公開できる状態にする。
- [ ] Scope:
  - Chrome Web Store申請に必要な `manifest` 項目の整備（公開前提の値確認を含む）
  - 配布パッケージ（zip）作成手順の整備
  - ストア掲載情報の下書き（短い説明/詳細説明/カテゴリ/サポート情報）
  - ライセンス整備（プロジェクトライセンス明示と依存ライセンス確認フロー）
  - 公開運用ドキュメント（提出チェックリスト、公開手順、リリース時の更新手順）
- [ ] Out of scope:
  - ストア審査の通過保証
  - マーケティング用素材の制作最適化（高品質バナー作り込みなど）
  - 新機能開発
- [ ] Assumptions:
  - 既存機能は公開可能な品質に到達しており、PBIでは公開準備の整備を主対象とする
  - 公開対象はChrome拡張（Manifest V3）
- [ ] Risks:
  - 権限説明不足や掲載情報不足による審査差し戻し
  - 依存ライセンス確認漏れによる公開リスク

# Acceptance Criteria

- [ ] AC1: 公開準備ドキュメントに従って、申請に必要な情報を一通り埋められる。
- [ ] AC2: `manifest` と配布パッケージ手順が公開用として整備され、再現可能である。
- [ ] AC3: ライセンス表記と依存ライセンス確認手順が明文化されている。

# Notes

- 公開準備の正本ドキュメントを1箇所に集約し、次回アップデート時に使い回せる形にする。
- 申請時点で未確定の入力値（URL/サポートメール等）はプレースホルダで明示する。
