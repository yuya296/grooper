# Requirement

- [ ] Purpose / value: 多言語対応により、非日本語話者でもPopup/Optionsを理解して利用できるようにする。
- [ ] Scope:
  - UI文言をi18nリソースへ抽出（Popup/Options中心）
  - 初期対応言語: 日本語 `ja` / 英語 `en`
  - ブラウザ言語に応じた初期言語選択
  - 言語設定の明示選択UI（必要最小限）と永続化
- [ ] Out of scope:
  - 3言語目以降の翻訳整備
  - ドキュメント全体の多言語化
- [ ] Assumptions:
  - Chrome Extension の `i18n` 機構または同等の軽量実装で対応可能
  - 文言キー設計を先に固定すれば追加言語の拡張が容易
- [ ] Risks:
  - 文言抽出漏れで言語混在が発生
  - 文言長差異によるレイアウト崩れ

# Acceptance Criteria

- [ ] AC1: Popup/Optionsの主要文言が `ja` / `en` で切替表示できる。
- [ ] AC2: 初回表示時はブラウザ言語を基に選択され、ユーザー変更後はその設定を優先する。
- [ ] AC3: 既存操作（保存、ルール編集、バリデーション表示）の機能回帰がない。

# Notes

- 文言キーと命名規則を先に決め、後続機能追加時の翻訳漏れを防ぐ。
- 運用ルール:
  - 新規UI文言は直書きせず `src/extension/i18n.ts` のキーへ追加する。
  - キーは `popup.*` / `options.*` / `common.*` / `language.*` の名前空間を使う。
  - 文字列補間が必要な場合は `{name}` 形式のプレースホルダを使う。
